# FEATURE Workflow

1. Codex defines intended behavior and acceptance criteria.
2. Search project memory and inspect impact boundaries.
3. Use an explorer when callers or dependencies are unclear.
4. Use an architecture agent when interfaces, persistence, or module boundaries change.
5. Delegate one bounded implementation package.
6. Perform independent review.
7. Verify acceptance criteria and regressions.
8. Synchronize project control files and memory.
9. Codex assigns final status.
