# REFACTOR Workflow

1. Codex defines behavior that must remain unchanged.
2. Search project memory for prior architecture decisions and rejected approaches.
3. Map callers, dependencies, and persistence boundaries.
4. Define a bounded migration plan.
5. Delegate implementation in small reversible steps.
6. Review for semantic drift and hidden compatibility changes.
7. Run regression verification.
8. Update architecture, control files, and confirmed project memory.
9. Codex assigns final status.
